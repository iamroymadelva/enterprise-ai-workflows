# Conversation Starter Philosophy

The starters are intentionally:
- short
- operational
- workflow-oriented
- easy to scan
- beginner-friendly

The onboarding experience should:
- reduce friction
- improve usability
- guide users progressively
- support structured workflows

---

# Workflow Guidance

SupportOps Documentation Builder works progressively instead of generating documentation immediately.

The workflow includes:
1. Language Selection
2. Documentation Type
3. Audience Definition
4. Platform / Environment
5. Documentation Goal
6. Required Sections
7. Formatting Requirements
8. Additional Context
9. Draft Validation
10. Final Documentation

---

# Help Menu Philosophy

Help interactions should remain:
- concise
- operational
- easy to scan
- workflow-focused

The framework should avoid:
- excessive onboarding
- long theoretical explanations
- generic AI-style guidance

The objective is to:
- improve usability
- improve workflow clarity
- support fast workflow initiation.
