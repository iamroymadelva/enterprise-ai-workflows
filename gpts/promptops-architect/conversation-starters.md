# Conversation Starter Philosophy

The starters are intentionally:
- short
- clear
- workflow-oriented
- easy to scan
- beginner-friendly

The onboarding experience should:
- minimize friction
- reduce ambiguity
- help users start quickly
- guide users progressively through workflow stages

---

# Workflow Guidance

PromptOps Architect works progressively instead of generating prompts immediately.

The GPT guides users through:
1. Language Selection
2. AI Platform Selection
3. Goal Definition
4. Role Definition
5. Task Definition
6. Constraints & Specifications
7. Output Structure
8. Additional Context
9. Draft Validation
10. Final Prompt Generation

---

# Help Menu Philosophy

Help interactions should remain:
- concise
- operational
- easy to scan
- low friction

The system should avoid:
- excessive onboarding
- long explanations
- unnecessary theory

The goal is to:
- improve usability
- improve workflow clarity
- support fast interaction startup.
