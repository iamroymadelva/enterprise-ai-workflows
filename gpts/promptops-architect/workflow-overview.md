# PromptOps Architect — Workflow Overview

## Purpose

PromptOps Architect uses a structured multi-stage workflow system instead of immediate prompt generation.

The workflow is designed to:
- reduce ambiguity
- improve instruction quality
- improve output consistency
- improve AI usability
- support reusable AI workflows

---

# Workflow Architecture

```text
User Intent
    ↓
Language Selection
    ↓
AI Platform Selection
    ↓
Goal Definition
    ↓
Role Definition
    ↓
Task Definition
    ↓
Constraints & Specifications
    ↓
Output Structure Definition
    ↓
Additional Context
    ↓
Prompt Draft Generation
    ↓
Validation & Refinement
    ↓
Final Prompt
```

---

# Workflow Stages

## 1. Language Selection

The system asks the user which language should be used during the session.

The selected language must remain consistent throughout:
- menus
- workflows
- generated prompts
- refinements
- recommendations

The language may only change if explicitly requested by the user.

---

## 2. AI Platform Selection

The system identifies:
- which AI platform will use the prompt
- platform-specific limitations
- platform-specific behaviors

Examples:
- ChatGPT
- Claude
- Gemini
- Cursor
- Copilot
- Midjourney
- Custom AI systems

---

## 3. Goal Definition

The system identifies the high-level objective.

Examples:
- analyze
- optimize
- summarize
- automate
- explain
- generate

The GOAL defines:
- the strategic purpose
NOT:
- the specific task.

---

## 4. Role Definition

The workflow defines:
- expertise level
- communication style
- analytical behavior
- operational perspective

Examples:
- recruiter
- engineer
- strategist
- technical writer
- support analyst

Role engineering improves:
- AI behavior consistency
- output quality
- workflow alignment

---

## 5. Task Definition

The TASK defines the specific action.

Examples:
- analyze resumes
- generate SOPs
- optimize workflows
- troubleshoot incidents
- create documentation

Separating:
- GOAL
from:
- TASK

improves prompt clarity significantly.

---

## 6. Constraints & Specifications

The workflow defines:
- formatting rules
- response depth
- forbidden behaviors
- tone
- verbosity
- technical level
- structure expectations

Constraints improve:
- consistency
- precision
- predictability

---

## 7. Output Structure Definition

The system identifies how the final output should be organized.

Examples:
- Markdown
- JSON
- HTML
- Reports
- Bullet points
- SOP format
- Workflow structures

Custom output formats must always be supported.

---

## 8. Additional Context

The workflow collects:
- examples
- references
- source material
- datasets
- screenshots
- templates
- existing prompts

This improves:
- context completeness
- AI alignment
- output quality

---

## 9. Prompt Draft Generation

The system generates:
# PROMPT DRAFT

before generating the final version.

This allows:
- refinement
- validation
- restructuring
- additional constraints
- iterative optimization

---

## 10. Validation & Refinement

The workflow supports:
- prompt refinement
- structural improvements
- technical enhancement
- simplification
- workflow expansion
- platform adaptation

Prompt engineering should always be iterative.

---

PromptOps Architect treats prompt engineering as:
- workflow architecture
NOT:

The system prioritizes:
- reusable workflows
- operational AI usability
- instruction precision
- context engineering
- scalable AI interaction systems
